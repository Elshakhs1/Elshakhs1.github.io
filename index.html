E‑Commerce Sales & Inventory Reporting
SQL MySQL Workbench Data Modelling

📌 Business Problem

An e‑commerce retailer had no structured visibility into sales by category, region, or month. Inventory stockouts caused missed revenue but were not tracked systematically. Reports relied on manual spreadsheet exports.

⚙️ Technical Approach

  • Designed a normalised relational schema: customers, products, orders, order_items, inventory.
  • Wrote analytical SQL queries for revenue segmentation, cohort retention, top SKUs, and inventory depletion.
  • Created a reorder alert system using 90‑day moving average daily sales.

📄 Key SQL Snippet – Monthly Revenue & AOV

🔍 View query
SELECT DATE_FORMAT(o.order_date, '%Y-%m') AS month, COUNT(DISTINCT o.order_id) AS total_orders, SUM(oi.quantity * oi.unit_price) AS gross_revenue, ROUND(SUM(oi.quantity * oi.unit_price) / COUNT(DISTINCT o.order_id), 2) AS avg_order_value FROM orders o JOIN order_items oi ON oi.order_id = o.order_id WHERE o.status = 'completed' GROUP BY month ORDER BY month;

📊 Outcomes & Insights

✅ Identified top 3 categories driving 58% of revenue.
✅ Flagged 8 SKUs with stock below 14 days of demand → enabled proactive reordering.
✅ Provided foundation for the Tableau dashboard below.

Customer Support KPI Analysis
Python pandas matplotlib

📌 Business Problem

The support team had no visibility into resolution times or CSAT by issue type. Monthly manual reporting from CSV exports took ~4 hours and lacked standard metrics like First‑Contact Resolution (FCR).

⚙️ Technical Approach

  • Ingested 1,200 tickets, cleaned data, engineered features: fcr_flag, sla_breach, handle_band.
  • Built a 4‑panel matplotlib dashboard showing monthly FCR/CSAT, CSAT by issue type, handle‑time distribution, and SLA breach by agent.
  • Automated the entire reporting pipeline – one script regenerates everything.

📄 Key Python Snippet – FCR & monthly aggregation

🔍 View code
df['fcr_flag'] = (df['reopened'] == 0).astype(int)
monthly = df.groupby('month').agg(
fcr_rate=('fcr_flag', 'mean'),
avg_csat=('csat_score', 'mean')
).reset_index()
monthly['fcr_rate'] = monthly['fcr_rate'] * 100

Co‑Working Space Occupancy & Revenue Analysis
Excel Python (openpyxl) Business Modelling

📌 Business Problem

The co‑working space lacked a monthly view of occupancy, cancellation trends, and revenue by membership tier. Pricing decisions were made without evidence of seasonal demand.

⚙️ Technical Approach

  • Consolidated 12 months of booking logs into a structured Excel workbook.
  • Built formula‑driven “Monthly Summary” using COUNTIFS/SUMIFS referencing raw data.
  • Automated chart refresh with a Python (openpyxl) script that generates pre‑formatted summary sheets.

📄 Example Excel Formula – Cancellation Rate

🔍 View formula
=IF(B3=0,0,C3/B3)
Where B3 = total bookings, C3 = cancelled bookings

📈 Outcomes

📉 Identified Q1 and Q3 utilisation dips → launched 15% off‑peak promo.
🚫 Cancellation rate visibility → new 48‑hour notice policy reduced cancellations by ~18%.
🤖 Python script cut manual reporting prep time from 2h → 10min.
